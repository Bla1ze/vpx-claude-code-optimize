sbbp.vbs Optimization Changes
==============================

All changes target hot paths (timer subs, per-frame loops, per-tick audio functions).
Init code, event handlers, and one-shot subs were left untouched.


FIX 1: FLIPPER TIMER 1ms → 10ms + SHARED GetBalls + PRESS GUARD
-----------------------------------------------------------------
Location: Line ~7528 (RightFlipper.timerinterval), RightFlipper_timer sub
What changed:
  - RightFlipper.timerinterval = 1  →  10
  - FlipperNudge now called with shared fBOT array (single GetBalls per tick)
  - Added press guard: If LFPress = 0 And RFPress = 0 Then Exit Sub
    Skips FlipperNudge entirely when both flippers are at rest.
  - FlipperNudge signature changed: added BOT as first parameter, removed internal GetBalls call.
Why: VPX renders at 60fps. Flipper correction at 100Hz is visually identical to 1000Hz.
Saves: ~4,500 function calls/sec. When flippers at rest: eliminates 100% of GetBalls + FlipperNudge.
       GetBalls reduced from 2 per tick (2000/sec at 1000Hz) to 1 per tick (100/sec at 100Hz).


FIX 2: FlipperTricks — CACHE startangle AND currentangle
----------------------------------------------------------
Location: Line ~7768 (Sub FlipperTricks)
What changed:
  - Cached Flipper.startangle into local `sa`, Abs(sa) into `absSa`
  - Cached Abs(Flipper.currentangle) into local `ca`
  - All subsequent reads use cached locals instead of COM property reads
Before:
  Dir = Flipper.startangle/Abs(Flipper.startangle)
  If Abs(Flipper.currentangle) > Abs(Flipper.startangle) - 0.05 Then
After:
  Dim sa: sa = Flipper.startangle
  Dim Dir: Dir = sa / Abs(sa)
  Dim ca: ca = Abs(Flipper.currentangle)
  Dim absSa: absSa = Abs(sa)
  If ca > absSa - 0.05 Then
Why: startangle is constant but still a COM read; currentangle read 3x per call.
Saves: ~10,000+ COM reads/sec at original 1000Hz (still ~1000/sec at 100Hz).

FIX 4: FlipperTrigger — CACHE ALL FLIPPER PROPS + INLINE TRIG
---------------------------------------------------------------
Location: Line ~7680 (Function FlipperTrigger)
What changed:
  - Cache Flipper.x, .y, .currentangle, .Length into locals fx, fy, fca, flen
  - Compute frad = (fca + 90) * PIover180 once
  - Inline Atn2 call with cached locals instead of calling AnglePP(Flipper.x, Flipper.y, ...)
  - Inline Distance with dx*dx + dy*dy instead of calling Distance(ballx,bally,Flipper.x,Flipper.y)
  - DistanceFromFlipper also updated to cache flipper props and use PIover180
Before (across FlipperTrigger + DistanceFromFlipper + AnglePP + Distance):
  6-10 COM reads of Flipper.x/.y/.currentangle per call
After:
  4 COM reads total (fx, fy, fca, flen), rest are locals
Saves: ~30,000 COM reads/sec at 1000Hz with 4+ balls (still ~3,000/sec at 100Hz).


FIX 5: Distance — ^2 → x*x
-----------------------------
Location: Line ~7666 (Function Distance)
What changed:
  Before: Distance = SQR((ax - bx)^2 + (ay - by)^2)
  After:  Dim dx, dy: dx = ax - bx: dy = ay - by
          Distance = SQR(dx * dx + dy * dy)
Why: ^ operator calls Exp(2*Log(x)) internally. Multiplication is a single FPU op.
Saves: ~8,000 exponentiation ops/sec (called from FlipperTrigger, FlipperDeactivate, etc.)


FIX 6: PRE-COMPUTED TRIG CONSTANTS
------------------------------------
Location: Line ~7595 (after PI declaration)
What changed:
  Added:  Dim PIover180: PIover180 = PI / 180
          Dim d180overPI: d180overPI = 180 / PI
  Replaced in dSin:    degrees * Pi/180  →  degrees * PIover180
  Replaced in dCos:    degrees * Pi/180  →  degrees * PIover180
  Replaced in Radians: Degrees * PI/180  →  Degrees * PIover180
  Replaced in AnglePP: * 180/PI          →  * d180overPI
Why: Eliminates 2 divisions per trig call.
Saves: ~8,000 divisions/sec from FlipperTrigger chain at full rate.


FIX 7: AUDIO FUNCTIONS — ELIMINATE ALL EXPONENTIATION
------------------------------------------------------
Location: Lines ~8725-8781 (AudioFade, AudioPan, Vol, Volz, BallVel, VolPlayfieldRoll, PitchPlayfieldRoll)

AudioFade/AudioPan (^10 → multiply chain):
  Before: AudioFade = Csng(tmp ^10)
  After:  Dim t2,t4,t8: t2=tmp*tmp: t4=t2*t2: t8=t4*t4
          AudioFade = Csng(t8 * t2)
  Negative branch uses same chain on -tmp, negates result.

BallVel (^2 → x*x + COM cache):
  Before: BallVel = INT(SQR((ball.VelX ^2) + (ball.VelY ^2)))
  After:  Dim vx, vy: vx = ball.VelX: vy = ball.VelY
          BallVel = INT(SQR(vx * vx + vy * vy))

Vol (^2 → x*x):
  Before: Vol = Csng(BallVel(ball) ^2)
  After:  Dim bv: bv = BallVel(ball): Vol = Csng(bv * bv)

Volz (^2 → x*x + COM cache):
  Before: Volz = Csng((ball.velz) ^2)
  After:  Dim vz: vz = ball.velz: Volz = Csng(vz * vz)

VolPlayfieldRoll (^3 → x*x*x):
  Before: VolPlayfieldRoll = RollingSoundFactor * 0.0005 * Csng(BallVel(ball) ^3)
  After:  Dim bv: bv = BallVel(ball)
          VolPlayfieldRoll = RollingSoundFactor * 0.0005 * Csng(bv * bv * bv)

PitchPlayfieldRoll (^2 → x*x):
  Before: PitchPlayfieldRoll = BallVel(ball) ^2 * 15
  After:  Dim bv: bv = BallVel(ball): PitchPlayfieldRoll = bv * bv * 15

BallSpeed (^2 → x*x + COM cache):
  Before: BallSpeed = SQR(ball.VelX^2 + ball.VelY^2 + ball.VelZ^2)
  After:  Dim vx, vy, vz: vx = ball.VelX: vy = ball.VelY: vz = ball.VelZ
          BallSpeed = SQR(vx * vx + vy * vy + vz * vz)

Why: ^ calls Exp(N*Log(x)). Multiply chains are 3-10x faster.
Saves: ~5,000+ exponentiation ops/sec across all audio functions.

FIX 15: LStateController.Update — MULTIPLE OPTIMIZATIONS
----------------------------------------------------------
Location: Lines 12181, 12894, 13070, 12981 (Class LStateController)

HasKeys: error-handling → .Count (Line 13070):
  Before: On Error Resume Next / o.Keys() / check Err.Number
  After:  HasKeys = (o.Count > 0)
  Why: Error handling is extremely expensive. .Count is a direct integer read.
       HasKeys is called 5 times per Update() tick.

AssignStateForFrame: Remove+Add → .Item in-place update (Line 12181):
  Before: m_currentFrameState.Remove key : m_currentFrameState.Add key, state
  After:  Set m_currentFrameState.Item(key) = state  (when key exists)
  Why: Remove+Add is 3 hash operations. .Item = is 1 hash operation.

CStr → CLng for color comparison (Line 12981):
  Before: If Not CStr(tmpLight.Color) = CStr(c) Then
  After:  If CLng(tmpLight.Color) <> CLng(c) Then
  Why: CStr allocates a new string per call. CLng is a stack-only integer conversion.
       This runs for every light with color data per frame.
  Saves: ~24,000 string allocations/sec at 60+ lights.

Cache m_on dictionary lookups (Line 12894):
  Before: (new FrameState)(lcItem.level, m_on(lightKey).Color, m_on(lightKey).Idx)
  After:  (new FrameState)(lcItem.level, lcItem.Color, lcItem.Idx)
  Why: lcItem already holds the same object. 2 redundant hash lookups per light per frame.


FIX 16: VR FLASHER FADES — ^1.5/^2/^2.5 → MULTIPLY CHAINS
-------------------------------------------------------------
Location: Lines ~13944-14118 (VRBGFLSP_1_timer, VRBGFLPA_1_timer, VRBGFLBump1_1_timer, VRBGFLBump2_1_timer)
What changed:
  Pre-compute power values into locals at top of each timer sub:
    sp_s2 = sp_s * sp_s           (replaces ^2)
    sp_s15 = sp_s * Sqr(sp_s)     (replaces ^1.5)
    sp_s25 = sp_s2 * Sqr(sp_s)    (replaces ^2.5)
  All 4 VR flasher timer subs updated with same pattern.
Before: 4-5 separate ^1.5/^2/^2.5 calls per timer tick per flasher
After: 1 Sqr + 2-3 multiplications, reused across all objects
Why: ^1.5 calls Exp(1.5*Log(x)). Sqr + multiply is much cheaper.
Saves: ~500-2,000 exponentiation ops/sec (only active in VR mode, self-disabling timers).


BUGFIX: DynamicBSUpdate gBOT → GetBalls
-----------------------------------------
Location: Line ~6922 (Sub DynamicBSUpdate)
What changed:
  - Restored BOT = GetBalls instead of BOT = gBOT
Why: DynamicBSUpdate runs from FrameTimer_Timer (display frame rate), NOT GameTimer_Timer.
gBOT is only updated in GameTimer_Timer (10ms). Between ticks, ball objects could be
invalidated (drained/destroyed), causing COM errors when accessing .X/.Y/.Z on stale refs.
The other two gBOT consumers (RollingUpdate, UpdateBallBrightness) are called from
GameTimer_Timer, so gBOT is correct for them.


BUGFIX: CoRTracker dynamic array resizing
-------------------------------------------
Location: Line ~7994 (Class CoRTracker, Sub Update)
What changed:
  - Restored dynamic ReDim Preserve when ball ID exceeds UBound(ballvel)
  - Removed fixed `if bid <= tnob` guard that silently dropped high-ID balls
Why: VPX ball IDs increment and are never reused. After several drains and launches,
IDs exceed tnob (5). The Dampener class accesses cor.ballvel(aBall.id) without bounds
checking, causing "Subscript out of range" at runtime.


FIX 17: DISTANCE-SQUARED GATING IN DynamicBSUpdate
----------------------------------------------------
Location: Line ~6991 (inner source loop)
What changed:
  - Pre-compute falloffSq = falloff * falloff and invFalloff = 1/falloff at sub entry
  - Compute dx = bx - sx, dy = by - sy, dSq = dx*dx + dy*dy for each ball-source pair
  - Gate with dSq < falloffSq instead of calling DistanceFast for every pair
  - DistanceFast only called for pairs that pass the gate (reuses cached dx, dy)
  - All /falloff divisions replaced with *invFalloff (3 instances)
Before: DistanceFast called for every ball-source pair, /falloff division per shadow
After: Simple multiply+compare gate eliminates most DistanceFast calls; invFalloff
       replaces division with multiplication for within-range shadows
Saves: With 5 balls × 4 sources, eliminates ~15 DistanceFast calls/frame (those out of
       range). At 60fps = ~900 DistanceFast calls/sec + ~180 divisions/sec eliminated.
