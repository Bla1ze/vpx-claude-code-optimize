Iron Maiden (Original 2022) VPW 1.0.12 — Applied Optimization Changes
======================================================================
Format: what changed, where, why, quantified savings.

---

1. FLIPPERTRICKS 1ms → 10ms (line ~21572)
   Changed `RightFlipper.timerinterval = 1` to `RightFlipper.timerinterval = 10`.
   Reduces FlipperTricks + FlipperNudge call frequency from 1000/sec to 100/sec.
   Eliminates ~3,600 function calls/sec (2 FlipperTricks + 2 FlipperNudge × 900 saved calls).
   Single biggest per-tick win on this table.

2. AUDIOFADE/AUDIOPAN ^10 → REPEATED SQUARING (lines ~22118, ~22140)
   Replaced `tmp ^10` with `t2=tmp*tmp : t4=t2*t2 : t8=t4*t4 : result=t8*t2`
   (3 multiplications instead of generic VBS exponentiation dispatch).
   Handled both positive and negative branches separately.
   At 60Hz × 3 balls: ~360 exponentiation dispatches/sec eliminated.

3. BALLVEL — COM CACHING + ^2 → MULTIPLY (line ~22154)
   Cached `ball.VelX` and `ball.VelY` into locals `vx`, `vy`.
   Replaced `(ball.VelX)^2 + (ball.VelY)^2` with `vx*vx + vy*vy`.
   Eliminates 2 redundant COM reads per call and generic ^2 dispatch.

4. BALLSPEED — COM CACHING + ^2 → MULTIPLY (line ~21489)
   Cached `ball.VelX`, `.VelY`, `.VelZ` into locals.
   Replaced `^2` with multiplication.
   Eliminates 3 COM reads per call.

5. VOL/VOLZ/VOLPLAYFIELDROLL/PITCHPLAYFIELDROLL — ^2/^3 → MULTIPLY (lines ~22159-22190)
   Vol: `BallVel(ball)^2` → `v = BallVel(ball) : Vol = Csng(v * v)`
   Volz: `ball.velz^2` → `vz = ball.velz : Volz = Csng(vz * vz)`
   VolPlayfieldRoll: `BallVel(ball)^3` → `bv*bv*bv`
   PitchPlayfieldRoll: `BallVel(ball)^2` → `bv*bv`
   Each also caches BallVel result to avoid redundant computation.

6. COMPLETE ROLLINGUPDATE REWRITE (line ~3984)
   - Pre-built `BallRollStr(19)` = "BallRoll_0".."BallRoll_19" and
     `MetalRollStr(19)` = "fx_metalrolling0".."fx_metalrolling19" at script load.
   - Cached `BOT(b).X`, `.Y`, `.Z`, `.VelX`, `.VelY` into locals.
   - Inline BallVel computation (eliminated function call overhead).
   - Inline AudioPan/AudioFade with ^10 repeated squaring.
   - Guarded `aBallShadow(b).Visible` writes with change detection.
   At 60Hz × 3 balls: ~1,800 COM reads/sec, ~540 BallVel calls/sec,
   ~720 string allocs/sec eliminated.

7. REALTIME_TIMER — PRE-CACHED FLIPPER STARTANGLES + CURRENTANGLE LOCALS (line ~6947)
   Pre-cached `RightFlipper.StartAngle` etc. into module-level constants at script load.
   Cached `RightFlipper.currentangle` etc. into locals — each value read once,
   used by 2 prim assignments.
   At 60Hz: ~8 COM reads/sec eliminated (4 StartAngle reads + 4 duplicate currentangle reads).

8. FLASHFLASHER — CACHED OBJLEVEL + PRE-COMPUTED POWERS + PRE-BUILT MATERIAL STRINGS (line ~7050)
   Cached `ObjLevel(nr)` into local `ol` (was read 6× per call).
   Pre-computed `ol2 = ol*ol`, `ol3 = ol2*ol`.
   Replaced `ObjLevel(nr)^2.5` with `ol2 * Sqr(ol)`.
   Replaced `ObjLevel(nr)^3` with `ol3`, `ObjLevel(nr)^2` with `ol2`.
   Pre-built `FlasherMatStr(20)` = "Flashermaterial0".."Flashermaterial20" at script load.
   Per flasher per tick: 4 exponentiation dispatches eliminated, 5 array derefs eliminated,
   1 string alloc eliminated.
   At 100Hz × 3 active flashers: ~1,200 ^n dispatches/sec + ~300 string allocs/sec eliminated.

9. LAMPFILTER ^1.6 → LOOKUP TABLE (line ~7916)
   Pre-computed `LampFilterLUT(100)` array with 101 entries for 0.00-1.00 range.
   `LampFilter(aLvl)` now does a table lookup instead of `^1.6` per call.
   LampFilter is called from LampFader.Update via cFilter callback for every active lamp.
   At ~140 lamps × 100Hz: up to ~14,000 ^1.6 dispatches/sec eliminated.

10. GI FADE — MODULE-LEVEL OPACITY VARIABLE (line ~3549)
    Replaced `g01_gion.Opacity = g01_gion.Opacity - 2` (read-modify-write, 2 COM ops per object)
    with module-level `GIFadeOpacity` variable updated once, then written to all 17 objects.
    tmrGIFade had 17 objects × 2 COM ops = 34 COM ops per tick.
    Now: 17 writes only = 17 COM ops per tick. 50% reduction.
    At fade rate ~50 ticks per full fade: ~850 COM reads eliminated per fade cycle.

11. LAMPFADER IsArray PRE-CACHE (LampFader class + line ~7170)
    Added `IsArr(140)` public boolean array to LampFader class.
    Set `IsArr(aIdx) = True` in MassAssign when array-type objects are stored.
    Replaced `IsArray(Lampz.obj(idx))` calls in LampTimer1_Timer, Update(), and
    TurnOnStates() with the pre-cached `IsArr(idx)` / `Lampz.IsArr(idx)` flag.
    IsArray is a VBScript runtime function that inspects the Variant type tag —
    the pre-cached boolean avoids this per-lamp per-tick.
    At ~77 lamps × 100Hz in LampTimer1: ~7,700 IsArray checks/sec eliminated.
    At ~140 slots × frame rate in Update: additional ~8,400+ IsArray checks/sec eliminated.

12. FLIPPERNUDGE — SHARED GETBALLS + COM CACHING (line ~21575, ~21584)
    Moved `GetBalls` call from inside FlipperNudge (called 2× per tick) to
    RightFlipper_timer (called once), passing the result to both FlipperNudge calls.
    Eliminates 1 redundant `GetBalls` COM call per tick (100/sec at 10ms interval).
    Cached `Flipper1.currentangle` into local `f1ca` (was read 2× per call).
    Cached `BOT(b).x`, `BOT(b).y` into locals in the ball loops.

13. FLIPPERTRICKS — COM CACHING (line ~21757)
    Cached `Flipper.startangle` and `Flipper.currentangle` into locals `fsa`, `fca`.
    Each was read 2-3× per call via COM boundary.
    At 100Hz × 2 calls: ~400-600 COM reads/sec eliminated.

14. DISTANCE ^2 → MULTIPLY (line ~21646)
    Replaced `(ax - bx)^2 + (ay - by)^2` with `dx*dx + dy*dy` using cached locals.
    Distance is called from FlipperTrigger and DistancePL (hot FlipperNudge path).

15. LAMPFADER Update() — CACHE Lvl*Mult + HOIST cFilter + FIX Mult BUG (line ~7915)
    Cached `modLvl = Lvl(x) * Mult(x)` once per lamp per frame — was computed 2-4× per lamp.
    For array-path with UseFunction: hoisted `filtLvl = cFilter(modLvl)` before the for-each loop.
    Previously cFilter was called per-object-in-array (2-3× per lamp) with identical arguments.
    BUG FIX: Non-UseFunction single-lamp path was using `Lvl(x)` without `Mult(x)`.
    Removed TypeName() debug check (2 runtime type inspections per lamp per frame).
    At 140 lamps × 60Hz: ~8,400 redundant multiplications + ~12,000 redundant cFilter calls/sec
    + ~16,800 TypeName checks/sec eliminated.

16. LAMPFADER CALLBACK — PRE-CACHED GetRef (line ~7820, ~7913)
    Pre-resolved `GetRef(procName)` during Callback property assignment, stored in `cCallbackRef(idx)`.
    Update() now calls `cCallbackRef(x) modLvl` directly instead of `Proc name & x, value`.
    Eliminates per-frame: 1 string concatenation (`name & x`) + 1 GetRef() lookup + 1 Proc() call
    per callback-enabled lamp.
    At ~30-40 callback lamps × 60Hz: ~1,800-2,400 GetRef lookups/sec + ~2,400 string allocs/sec eliminated.

17. LAMPTIMER1 — PRE-BUILT LIGHT INDEX ARRAY (line ~7163)
    After InitLampsNF, builds `LampzLightIndices()` and `LampzLightIsArr()` arrays containing
    only the indices where `IsLight = True`. LampTimer1_Timer now iterates ~30 light indices
    instead of all 140 lamp slots, skipping ~110 conditional checks per tick.
    At 62.5Hz (16ms timer): ~6,875 wasted iterations/sec eliminated.

18. REALTIME_TIMER — SPINNER CURRENTANGLE CACHING (line ~6975)
    Cached `Spinner001.currentangle` and `Spinner002.currentangle` into locals.
    Each was a COM read used once — minor but consistent with the flipper pattern.
    At 60Hz: ~120 COM reads/sec eliminated.

19. PUP OVERLAY CLEAR.PNG TYPO FIX (lines ~20589, ~20595)
    Changed `"PuPOverlays\\clear .png"` (with space before .png) to `"PuPOverlays\\clear.png"`.
    Also copied existing `clear1.png` → `clear.png` in PupOverlays/ and Backglass/ directories.
    Eliminates "Image not found" file I/O warnings on every PuP overlay clear during gameplay.

20. pUpdateScores — PRE-BUILT JSON STRINGS (line ~17999)
    Pre-built `pScoreOrangeJson(4)`, `pScoreWhiteJson`, `pScoreRedJson` at module level.
    Pre-built `pScorePosLabel(4)` = "ScorePos1".."ScorePos4" and `pScore4Json(4)` position variants.
    Cached `cpLabel = pScorePosLabel(CurrentPlayer)` once per call instead of `"ScorePos" & CurrentPlayer+1`.
    Eliminates all JSON string concatenation from the 250ms score update timer.
    At 4Hz: ~20+ string allocs/sec eliminated + reduced GC pressure during score flicker.
